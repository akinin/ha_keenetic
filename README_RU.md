# Интеграция Keenetic Router для Home Assistant

[English ver.](README.md)

Пользовательская интеграция Home Assistant для роутеров Keenetic. Интеграция работает через локальный REST API Keenetic и добавляет в Home Assistant состояние роутера, клиентов, интерфейсов, Wi-Fi, Mesh-узлов, USB/media, обновлений прошивки и базовые элементы управления.

## Возможности

- Диагностика роутера: загрузка CPU, память, uptime, hostname, domain name, внешний IP.
- Мониторинг интерфейсов: WAN/VPN подключение, статистика трафика, Ethernet-порты, Wi-Fi интерфейсы.
- Диагностика Wi-Fi радио: канал, ширина канала, битрейт и температура, если роутер отдает эти данные.
- Управление Wi-Fi: включение/выключение интерфейсов и настройка idle timeout клиентов.
- Работа с клиентами: device tracker и выбор политики доступа для выбранных или всех известных клиентов.
- Управление port forwarding: опциональные переключатели для правил проброса портов.
- Мониторинг Mesh: состояние Mesh-узлов и диагностические атрибуты.
- USB/media: переключатели питания USB, состояние media и сенсоры заполнения накопителей, если это поддерживается роутером.
- Обновление прошивки с опциональным backup перед установкой.
- Сервисы для прямого вызова Keenetic API и создания backup роутера.

## Установка

### Через HACS

1. Откройте HACS.
2. Перейдите в "Integrations".
3. Найдите "Keenetic Router".
4. Установите интеграцию.
5. Перезагрузите Home Assistant.

### Ручная установка

1. Скачайте архив последнего релиза.
2. Скопируйте папку `ha_keenetic` в `custom_components`.
3. Перезагрузите Home Assistant.

## Доступ к Keenetic

Интеграция подключается к web API роутера по HTTP или HTTPS, используя адрес, порт, логин и пароль, указанные в Home Assistant. По возможности используйте локальный адрес роутера и не открывайте веб-интерфейс за пределы доверенной сети.

Важно по безопасности: QR-коды Wi-Fi по своей природе содержат пароль от Wi-Fi сети. Не используйте для Wi-Fi тот же пароль, что и для учетной записи администратора Keenetic.

Для автоматического обнаружения роутера включите UPnP в веб-интерфейсе Keenetic:

## Настройка

Если UPnP включен, Home Assistant может обнаружить роутер автоматически.

![Обнаруженная интеграция](images/1.png)

Для ручной настройки:

1. Перейдите в "Settings" > "Devices & services".
2. Нажмите "+ ADD INTEGRATION".
3. Найдите "Keenetic Router".
4. Укажите адрес роутера, порт, логин, пароль и параметр SSL.

## Опции

После добавления интеграции откройте ее параметры. Настройки разделены на несколько экранов:

- Подключение: адрес роутера, порт, логин, пароль и режим SSL.
- Основное: интервал опроса и текущая диагностика роутера.
- Wi-Fi: QR-коды для всех или выбранных Wi-Fi сетей.
- Клиенты: политики доступа и device tracker для выбранных клиентов или всех клиентов.
- Функции и бэкапы: переключатели port forwarding и типы файлов backup.

Откройте нужные разделы, измените параметры, затем выберите "Сохранить" в меню настроек.

## Сущности

### Sensors

- CPU, память, uptime.
- Hostname и domain name.
- Внешний IP.
- Количество Wi-Fi клиентов.
- Состояние Mesh-узлов.
- Диагностика Ethernet, Wi-Fi, USB/storage и VPN, когда данные доступны.

### Binary Sensors

- Подключение к роутеру.
- Подключение WAN/VPN интерфейсов.
- Ping Check по интерфейсам.
- Состояние USB/media.

### Switches

- Wi-Fi интерфейсы.
- WAN/VPN и другие управляемые интерфейсы.
- Правила port forwarding.
- Публичный доступ к Web Configurator.
- Питание USB-портов.

### Numbers

- Idle timeout клиентов Wi-Fi для поддерживаемых Wi-Fi интерфейсов.

### Selects

- Политика доступа в интернет для выбранных клиентов.

### Device Trackers

- Опциональные трекеры для выбранных или всех известных клиентов.

### Update

- Обновление прошивки с опциональным backup конфигурации и прошивки.

## Сервисы

### `ha_keenetic.request_api`

Выполняет прямой запрос к Keenetic API. Полезно для диагностики и продвинутых автоматизаций.

### `ha_keenetic.backup_router`

Скачивает backup конфигурации и/или прошивки роутера.

## Поддерживаемые устройства

Проверено на:

- Keenetic Giga
- Keenetic Hero 4G
- Keenetic Sprinter SE

Другие модели и режимы Keenetic тоже должны работать, если нужные API-эндпоинты доступны.
